Polars is a [Python library][polars user guide], that has been in the back of my head for a while now. Yet I never really came around to tinker with it and test out it's possibilities. Until now.

Code will be available in a [dedicated repository][github repo].

## Philosophy

The very essence of Polars is speed. It aims to be lightning fast&trade; and to also handle huge datasets, much larger than the available RAM. How it does that is something we will come to, when we talk about Polars implementation of Lazy Processing, what in my opinion is the biggest feature distinguishing Polars from Pandas.

On why it's generally fast: Polars' core is written, in it's entirety in Rust. It also works in parallel utilizing all available resources and it uses Apache Arrow.

## Core Concepts

### Data Types

Polars implements [Apache Arrow's][apache arrow] [data types][apache arrow python data types], except for some exceptions, the `String`, `Categorical` and `Object` data types. As Apache Arrow is a platform that provides a language-independent in-memory format for columnar data, I definitely get, why Polars uses it.
